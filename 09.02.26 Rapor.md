# 09.02.26 Rapor

## Hata Mesaji

```text
VirtualizedList: You have a large list that is slow to update - make sure your renderItem function renders components that follow React performance best practices like PureComponent, shouldComponentUpdate, etc. {"contentLength": 2468.571533203125, "dt": 4181, "prevDt": 5588}
```

## Bu Ne Anlama Geliyor?

Bu bir performans uyarisidir. Cokme (crash) degil.
Liste guncellemeleri JS tarafinda yavas kaliyor demektir.

- `contentLength: 2468.5715...` -> listenin toplam icerik uzunlugu (px)
- `dt: 4181` -> bu guncelleme yaklasik 4.1 saniye surmus
- `prevDt: 5588` -> onceki guncelleme yaklasik 5.6 saniye surmus

## Olası Nedenler

- `renderItem` icinde pahali islem
- Itemlarin gereksiz yeniden render olmasi
- Inline fonksiyon/obje nedeniyle prop referanslarinin surekli degismesi
- Liste ust seviyesinde cok sik state degisimi

## Kisa Cozum Checklist

- Item componentlerini `React.memo` ile sar
- `renderItem`, handler ve style objelerini `useCallback` / `useMemo` ile stabilize et
- `keyExtractor` icin stabil `id` kullan
- `extraData`yi minimumda tut
- `windowSize`, `maxToRenderPerBatch`, `removeClippedSubviews`, `estimatedItemSize`/`getItemLayout` ayarlarini optimize et
- Dev modda asiri log varsa azalt (profiling degerlerini olumsuz etkileyebilir)
